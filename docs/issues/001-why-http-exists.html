The HTTP Mystery

Reading time: 30-45 min Topic: HTTP Message Result: Test yourself

Why does HTTP exist if TCP already sends bytes?

今天的核心不是背定义，而是抓住一个问题：字节到了以后，另一台电脑怎么知道这些字节代表什么？

Imagine a world with no HTTP. Your browser connects to a server. The server sends back a stream of bytes. Now what? Are those bytes HTML, JSON, an image, an error, a redirect, or a login result?

TCP can carry the bytes, but TCP does not explain the message. HTTP exists because browsers and servers needed a shared format. It gives meaning to the byte stream.

Every technology exists because of a problem. HTTP exists because the Web needed a common language between clients and servers.

Today's Answer

TCP solves transport. HTTP solves meaning.

Look For

  • Method
  • Path
  • Headers
  • Status code
  • Body

Why

HTTP answers questions TCP does not answer: what action is requested, which resource is targeted, what metadata is attached, whether the request succeeded, and what kind of content came back.

Without HTTP, every website could invent its own private format. Browsers, proxies, API clients, CDNs, and debugging tools would lose a common foundation.

History

HTTP was born with the early World Wide Web. Its first job was simple: let a client ask for a document and let a server return it.

The Web grew because many different machines could agree on a small readable protocol. A boring shared standard unlocked a huge ecosystem.

Trade-off

HTTP is not always the best protocol. Real-time two-way communication may use WebSocket. Internal high-performance service calls may use gRPC. Custom systems may use raw TCP.

HTTP wins when you need a general, readable, widely supported agreement.

Boss Challenge

Design the REST API. Explain method, path, success status code, and not-found behavior.

Test

  1. What problem does HTTP solve that TCP does not solve?
  2. Is POST /orders/123/delete good REST design? Explain.
  3. A server receives bytes over TCP. Why might it still need HTTP headers?
  4. Design an Order API for list, read, create, address update, and cancel.
  5. HTTP is stateless. How can a website keep a user logged in?
