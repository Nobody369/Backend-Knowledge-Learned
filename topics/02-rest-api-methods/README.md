# Topic 02: REST API Methods

Today we learned that HTTP methods carry design meaning.

Method map:

- `GET` reads a resource.
- `POST` creates a resource or triggers a creation workflow.
- `PUT` replaces a whole resource.
- `PATCH` updates part of a resource.
- `DELETE` removes or cancels a resource.

Boss challenge:

Design an Order API:

- List orders
- Read one order
- Create an order
- Update shipping address
- Cancel an order

Good REST design makes the resource visible in the path and the action visible in the method.
